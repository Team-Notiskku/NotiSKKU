## 📌 commit conventions
feat : 새로운 기능 추가   
fix : 버그 수정   
design : UI 디자인 변경   
style : 코드 포맷 변경, 코드 수정이 없는 경우   
refactor : 코드 리팩토링   
docs : 문서 수정, 주석 등   
test : 테스트 추가, 테스트 리팩토링   
chore : 패키지 설치, 빌드 설정 변경   
rename : 파일, 폴더명 수정   
ci : CI/CD 관련 설정 변경   
perf : 성능 개선   
revert : 이전 커밋을 되돌릴 때
